# Intent: Bitcoin Learning Graph

## Problem
People want a single map to understand Bitcoin learning paths, self-assess what they already know, and quickly spot prerequisite gaps.

## Outcome
Deliver a public static website (GitHub Pages) with an interactive hierarchical prerequisite graph for Bitcoin learning, including progress tracking and curated node resources.

## Core Constraints
- Static-only runtime (no backend/API in v1).
- Open collaboration through GitHub issues and PRs.
- Strict prerequisite semantics, but no hard blocking.
- Agile vertical slices: every commit should keep a visible working state.
- TDD and close-the-loop verification for all work.

## Success Criteria
- Users can explore graph, set progress states, inspect gaps, search, and filter.
- Site works on desktop and mobile.
- Site is deployed and reachable online via GitHub Pages.
- Content includes broad Bitcoin topics derived from trusted sources.
