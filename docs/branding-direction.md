# Branding Direction: The Bitcoin Map Room

## Context from product goals
This product is a public learning map, not a trading dashboard. The interface should feel like an atlas for serious study: exploratory, precise, and calm under complexity.

## Core brand idea
- Name for the direction: **Bitcoin Map Room**
- Concept: a modern cartography desk where people track what they know and what they still need to connect
- Emotional target: trustworthy, focused, and intellectually curious

## Visual pillars
1. Editorial + technical typography
- Display voice (`Syne`) creates identity and hierarchy.
- Reading voice (`Archivo`) keeps long descriptions comfortable.
- Utility voice (`Azeret Mono`) marks metadata, tags, and controls with precision.

2. Cartographic atmosphere
- Layered parchment/blueprint backgrounds with subtle grid overlays.
- Glass-like panels with structured borders to suggest “work surface” tools.
- Node map area treated as the active desk canvas.

3. Progress-as-signal
- Status controls are intentionally strong and high contrast.
- Gap states are warm alert blocks, not destructive errors.
- Overview counters make progress legible at a glance.

4. Distinctive signature
- The header behaves like a mission card: `Proof of Learning` + atlas title + live concept/branch counts.
- This differentiates the site from generic graph demos.

## Color direction
- Light mode: warm parchment neutrals + deep ink + teal/copper accents.
- Dark mode: deep navy blueprint + cyan/orange accents.
- Strict avoidance of generic purple-gradient SaaS aesthetics.

## Motion direction
- One meaningful load-in (`rise-in`) for major surfaces.
- Hover lift on controls and legend chips for tactile feedback.
- Mobile detail panel remains a bottom sheet with explicit drag-handle affordance.

## UX constraints preserved
- Static-site architecture only.
- Existing issue-based contribution flows remain primary.
- Search, filtering, progress marking, export/import, URL state, and theme toggle remain intact.
