# Leaf Node Audit (2026-03-10)

Current leaf nodes are topics with zero dependents in the graph after rebuilding on 2026-03-10.

This audit focused on whether any leaf looked obviously under-connected based on:
- how foundational the topic is inside Bitcoin itself
- whether another node in this repo already relies on the concept in practice
- whether the relationship is a strict prerequisite rather than a loose association

Changes applied in this audit:
- `fundamentals.diffie-hellman-key-exchange` is now a prerequisite for `privacy.silent-payments` and `protocol.v2-p2p-transport`.
- `custody.passphrase` now depends on `dev.bip39`.
- `lightning.route-blinding` now depends on `lightning.onion-routing`.
- `lightning.bolt12-offers` now depends on `lightning.route-blinding`.

Topics reviewed but intentionally left as leaves for now:
- `privacy.dandelion`: useful and specific, but no existing node currently requires it as a strict prerequisite.
- `extension.ark`: advanced and potentially foundational for future material, but no current node in the repo is clearly built on top of Ark yet.
